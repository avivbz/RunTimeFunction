תכנון מסלול טיול

תרשים של המסלולים האפשריים בין האתרים השונים

מהו המסלול הקצר ביותר?

זוהי בעיית "הסוכן הנוסע" (Traveling Salesman Problem) - בעיה מפורסמת במדעי המחשב. המטרה היא למצוא מסלול המתחיל בנקודה מסוימת, עובר בכל הנקודות פעם אחת בדיוק, וחוזר לנקודת ההתחלה, כך שהמרחק הכולל יהיה מינימלי. בעיה זו ידועה כבעיה NP-קשה, כלומר אין אלגוריתם יעיל (פולינומיאלי) הידוע לפתרון מדויק שלה עבור מקרה כללי.

מספר המסלולים

נניח וישנם 10 אתרים. מכל אתר ניתן להגיע לכל אתר אחר.

  • מנקודת המוצא, יש לכל היותר 9 אתרים שניתן להגיע.
  • נניח הגענו לנקודה a1. מהנקודה a1 יש לכל היותר 8 אתרים שניתן להגיע. נניח הגענו לנקודה a2.
  • מהנקודה a2 יש לכל היותר 7 אתרים. וכן הלאה...
  • כלומר מספר במסלולים האפשריים עבור 10 אתרים הוא: 9*8*7*6*5*4*3*2*1

מדידת יעילות

בשקף זה נלמד על מדידת יעילות של אלגוריתמים באמצעות ספירת פעולות יסוד.

דוגמה לפונקציית זמן ריצה של פעולה המחשבת סכום של k איברים במערך

public static int SumArray (int[] arr, int k)
{
    int sum = 0;
    for (int i = 0; i < k; i++)
        sum = sum + arr[i];
    return sum;
}

פעולת יסוד 1: int sum = 0; - פעולה אחת

פעולת יסוד 2: i < k - k+1 פעולות (k פעמים + בדיקה נוספת בסוף)

פעולת יסוד 3: sum = sum + arr[i]; i++ - 2k פעולות (2 פעולות × k פעמים)

סה"כ: 1 + (k+1) + 2k = 3k + 2 פעולות

כאשר k = n (גודל הקלט), פונקציית זמן הריצה היא: f(n) = 3n + 2

דוגמא 2: חיפוש במערך ממוין בסדר עולה

public static int Search1 (int[] arr, int item)
{
    int n = arr.Length;
    for (int i = 0; i < n; i++)
        if (arr[i] == item)
            return i;
    return -1;
}

פעולה: 3n+2 פעולות לכל היותר

מהי פונקציית זמן הריצה עבור מערך בגודל n, במקרה שהערך של item לא נמצא במערך?

במקרה הגרוע ביותר (כשהערך לא נמצא במערך), נצטרך לעבור על כל איברי המערך, ולכן פונקציית זמן הריצה תהיה O(n), כלומר 3n+2 פעולות.

ההסבר המפורט:

  • פעולה אחת להשמה: int n = arr.Length;
  • n+1 פעולות לבדיקת התנאי: i < n (n פעמים + בדיקה אחרונה שמפסיקה את הלולאה)
  • n פעולות להגדלת i: i++
  • n פעולות לבדיקת התנאי: arr[i] == item
  • פעולה אחת לביצוע: return -1;

סה"כ: 1 + (n+1) + n + n + 1 = 3n+3

לעתים נוהגים לכתוב 3n+2 כי לא סופרים את הפעולה האחרונה (return).

משימה

נתחו את היעילות של הפעולה:

הפעולה שלפניכם מקבלת שני מערכים שגודלם n. כל האיברים במערך הראשון שונים זה מזה. הפעולה תדפיס את כל המספרים מהמערך הראשון המופיעים במערך השני, וכמה פעמים הם מופיעים בו.

לשם כך, נעבור על כל איבר במערך הראשון ונבדוק האם הוא מופיע במערך השני ונמנה את מספר מופעיו.

public static void SharedNums(int[] arr1, int[] arr2)
{
    int counter;
    for (int i = 0; i < arr1.Length; i++)
    {
        counter = 0;
        for (int j = 0; j < arr2.Length; j++)
            if (arr1[i] == arr2[j])
                counter++;
        if (counter > 0)
            Console.WriteLine(arr1[i] + " appears " + counter + " times");
    }
}

מהי סיבוכיות זמן הריצה של הפעולה הזו?

סיבוכיות זמן הריצה של הפעולה היא O(n²).

ניתוח מפורט:

  • הלולאה החיצונית רצה n פעמים (גודל arr1).
  • בכל איטרציה של הלולאה החיצונית, הלולאה הפנימית רצה n פעמים (גודל arr2).
  • סך הכל, תנאי ה-if הפנימי נבדק n×n = n² פעמים.
  • פעולות נוספות כמו אתחול counter והדפסה הן O(n) במקרה הגרוע.
  • לכן, הסיבוכיות הכוללת היא O(n²).

ניתן לוותר על האתחול ולחסוך פעולה אחת

איך ניתן לשפר את הקוד כדי לחסוך את פעולת האתחול?

public static int Search1(int[] arr, int k)
{
    for (int i = 0; i < arr.Length; i++)
        if (arr[i] == k)

במקום לאחסן את אורך המערך במשתנה נפרד `n`, ניתן לגשת ישירות לתכונה `Length` של המערך בתנאי הלולאה. זה חוסך פעולת השמה אחת, אך סיבוכיות הזמן נשארת O(n).

הרעיון בפעולה זו הוא לוודא שבטוח הפעולה תמצא הערך:

  • נבדוק האם הערך שמחפשים נמצא בתא האחרון.
  • אם לא נשמור את הערך של התא האחרון. נשים במקומו את הערך שמחפשים.
  • לולאת while עוברת עד שנמצא הערך.
  • אם הערך נמצא בתא האחרון זה אומר שבמערך המקורי הערך שמחפשים לא היה.

מהי פונקציית זמן הריצה של פעולה זו עבור מערך בגודל n?

פונקציית זמן הריצה עדיין היא O(n), אך עם שיפור קבוע:

  • במקרה הגרוע ביותר, עדיין נצטרך לעבור על כל איברי המערך (O(n)).
  • השיפור הוא שלא צריך לבדוק תנאי יציאה בכל איטרציה (i < n), מכיוון שבוודאות נמצא את הערך (הרי שמנו אותו בסוף המערך).
  • זה חוסך n בדיקות של תנאי הלולאה, אבל סיבוכיות האלגוריתם נשארת O(n).

המימוש עשוי להיראות כך:

public static int ImprovedSearch(int[] arr, int k)
{
    int n = arr.Length;
    int last = arr[n-1]; // שמירת הערך האחרון
    arr[n-1] = k; // שמים את הערך המבוקש בסוף
    
    int i = 0;
    while (arr[i] != k)
        i++;
    
    arr[n-1] = last; // החזרת הערך המקורי
    
    if (i < n-1 || k == last)
        return i; // נמצא במערך המקורי
    else
        return -1; // לא נמצא במערך המקורי
}

מיון בועות

פעולה זו מקבלת מערך וממיינת את הערכים בו בשיטה הנקראת מיון בועות.

הפעולה עובדת עם שתי לולאות מקוננות התלויות באורך המערך

public static void BubbleSort(int[] arr)
{
    int n = arr.Length;
    for (int i = 0; i < n - 1; i++)
        for (int j = 0; j < n - i - 1; j++)
            if (arr[j] > arr[j + 1])
            {
                // החלפת איברים
                int temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
            }
}

מהי סיבוכיות זמן הריצה של מיון בועות?

סיבוכיות זמן הריצה של מיון בועות היא O(n²).

ניתוח מפורט:

  • הלולאה החיצונית רצה (n-1) פעמים.
  • הלולאה הפנימית רצה בפעם הראשונה (n-1) פעמים, בפעם השנייה (n-2) פעמים וכן הלאה.
  • סך הכל מספר האיטרציות הוא (n-1) + (n-2) + ... + 2 + 1 = n(n-1)/2.
  • לכן, סיבוכיות הזמן היא O(n²).

סיכום - סיבוכיות זמן ריצה

בשיעור זה למדנו:

  • כיצד מודדים יעילות של אלגוריתמים באמצעות ספירת פעולות יסוד
  • הבנו את המשמעות של התלות בגודל הקלט (n)
  • ראינו דוגמאות לאלגוריתמים בסיבוכיות שונה:
    • סיבוכיות O(n) - חיפוש ברשימה לא ממוינת
    • סיבוכיות O(n²) - מיון בועות, חיפוש איברים משותפים בשני מערכים
    • סיבוכיות O(n!) - בעיית הסוכן הנוסע
  • למדנו על שיפורים שניתן לבצע באלגוריתמים כדי להקטין את זמן הריצה
